The ZK pipeline

How it works

01
User balances (private)
Each user's balance stays secret — never revealed.
02
Merkle-sum tree → public root
Balances hash into a tree committing to their total.
03
RISC Zero zkVM
Proves reserves ≥ Σ balances, each balance in range.
04
Groth16 wrap
Compresses the proof into a 260-byte seal.
05
Soroban verify
Contract checks the proof + reads live USDC reserves.
06
On-chain attestation
Solvent ✓ — published for anyone to audit.
